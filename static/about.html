TV Thing

Stream over-the-air ATSC television to any device on the local network.

How It Works

Antenna | v +--------------------------+ +-------------------+ | Hauppauge HVR-1600 | | Web Browser | | (Conexant CX23418) | | | | | | +-------------+ | | DVB Frontend | | | Station | | | (Samsung S5H1409) | | | Dropdown | | | | | | +-------------+ | | | tunes to freq | | +-------------+ | | v | | | | | | Internal ATSC Decoder | | | HLS Video | | | | | | Player | | +-----------+--------------+ | | (hls.js) | | | | +-------------+ | | DVB TS data +--------^----------+ v | +------------------+ HTTP requests | cvlc | | | (VLC headless) | +--------+----------+ | | | Python Server | | DVB tuning + | | (port 8080) | | TS demux + | | | | H.264 transcode | | /api/channels | | + HLS output | | /api/tune | | | | /api/stop | | all in one | | /api/status | | process! | | /stream/*.m3u8 | | | | /stream/*.ts | +--------+----------+ +---------^---------+ | | +------> .m3u8 + .ts segments --+ in streams/ directory

The signal path has three stages:

  1. Tuning, capture, and transcoding — cvlc (headless VLC) handles everything in one process: it commands the DVB frontend to lock onto an ATSC frequency, demuxes the transport stream, selects the right program, transcodes video to H.264 and audio to AAC, and writes HLS segments directly to disk.
  2. Serving — The Python server serves the HLS playlist and segments to any browser on the LAN.
  3. Playback — hls.js in the browser fetches segments and plays them back in real time.

Known Quirks

  • The cx18 driver's DVR device (/dev/dvb/adapter0/dvr0) never produces data despite successful frontend lock. VLC's internal DVB implementation bypasses this entirely.
  • Only one channel can be tuned at a time (single tuner).
  • Channel changes take ~8–10 seconds (VLC startup + tune + lock + first HLS segment).
  • Subchannels on the same frequency share one RF channel. VLC's --program flag selects the specific subchannel by service ID.
