TV Thing

Stream over-the-air ATSC television to any device on the local network.

How It Works

Antenna | v +--------------------+ | Hauppauge HVR-1600 | cx18 kernel driver | DVB frontend | /dev/dvb/adapter0/frontend0 (tuning) | /dev/dvb/adapter0/ | /dev/dvb/adapter0/demux0 (section filters) +--------+-----------+ NOTE: dvr0 is broken on this card — VLC works around it | | (tuned, locked) | +-----+-------------------------------+ | | v v +------------------+ +--------------------+ | cvlc | | fetch_vct.py | (during rescan only) | atsc://frequency | | | | DVB tune + demux | | opens demux0 fd | | select --program | | DMX_SET_FILTER: | | H.264 + AAC | | PID 0x1FFB (PSIP)| | livehttp HLS out | | table 0xC8 (TVCT)| +--------+---------+ | reads virtual ch # | | | writes vct_data.json v +--------------------+ streams/live.m3u8 streams/segment_NNN.ts | v +------------------+ | server.py | Python HTTP, ThreadingMixIn, port 8080 | /api/channels | serves channels.json | /api/tune | spawns cvlc, manages process | /api/stop | kills cvlc, clears streams/ | /api/status | returns streaming state | /api/rescan/ | SSE: w_scan → fetch_vct → parse_channels | stream | | /stream/*.m3u8 | serves HLS playlist | /stream/*.ts | serves HLS segments +--------+---------+ | v +------------------+ | Browser (hls.js) | MSE-based HLS playback, any device on LAN | Channel list | PSIP virtual numbers (4.1, 5.1, 9.1 …) | Quality picker | 480p / 720p / 1080p +------------------+

The signal path has four stages:

  1. Tuning & streaming — cvlc (headless VLC) commands the DVB frontend, demuxes the transport stream, selects the right program by service ID, transcodes video to H.264 and audio to AAC, and writes HLS segments to streams/.
  2. Channel numbering — During a rescan, fetch_vct.py opens a separate fd on demux0 and installs a DVB section filter for PID 0x1FFB (the PSIP base PID) to read the Virtual Channel Table. This gives real virtual channel numbers (WRC-TV = 4.1, Fox 5 = 5.1) rather than RF channel numbers.
  3. Serving — The Python server serves the HLS playlist and segments, plus the REST API and web UI.
  4. Playback — hls.js in the browser fetches segments and plays them in real time.

Channel Rescan

The rescan page runs a two-phase process:

  1. w_scan — scans all ATSC frequencies (RF channels 2–36) and finds active stations. Progress streams live to the browser via Server-Sent Events.
  2. VCT fetch — for each found frequency, fetch_vct.py tunes briefly with VLC then reads the PSIP Virtual Channel Table via a DVB section filter on /dev/dvb/adapter0/demux0. This extracts real virtual channel numbers (major.minor) broadcast by each station.

The channel list is rebuilt with correct PSIP numbers after both phases complete. Tuning is blocked (returns 409) while a scan is in progress.

Known Quirks

  • The cx18 driver's DVR device (/dev/dvb/adapter0/dvr0) never produces data despite a successful frontend lock — a known driver issue with this card. VLC's internal DVB stack works around this entirely and is the only path that produces data.
  • DVB section filters (used by fetch_vct.py) work fine even though DVR doesn't — they go through the demux device directly rather than through dvr0.
  • Only one channel can be tuned at a time (single tuner).
  • Channel changes take ~8–10 seconds (VLC startup + tune + lock + first HLS segment).
  • Subchannels on the same RF frequency share one multiplex. VLC's --program flag selects the specific subchannel by PSIP service ID.
  • RF channel 7 (177 MHz) does not produce a signal at this location and is excluded from the channel list.
